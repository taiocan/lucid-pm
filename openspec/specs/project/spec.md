## Purpose

Extracts structured items from unstructured text using a project schema and an external AI model
(Gemini).

## Entities

### ExtractedItem

> `modules/pm_structuring/src/extractor.rs`

Represents a single, structured work item (such as a user story, task, or bug) parsed from an
unstructured text source using an AI model. Its structure and vocabulary are validated against a
project-specific schema.

**Properties:**

| Name | Type | Description |
|------|------|-------------|
| title | string | The summary or name of the work item. |
| description | string | The detailed body of the work item, parsed from the source text. |
| type | string | The classification of the work item (e.g., 'Feature', 'Bug'), which must conform to the project's schema. |
| status | string | The lifecycle state of the work item (e.g., 'To Do', 'Done'), which must conform to the project's schema. |
| parent_id | string | The identifier of the parent item, allowing for hierarchical structuring. |

**Relationships:**

- belongs to ExtractedItem (An item can be a sub-item of another, creating a work breakdown structure.)

## Requirements

### Requirement: ExtractedItemValidation

The system SHALL validate ExtractedItem according to these rules:
- Item 'type' must be a value defined in the ProjectSchema.
- Item 'status' must be a value defined in the ProjectSchema.

#### Scenario: ExtractValidWorkItem
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** A source text describing a work item and a ProjectSchema defining its type and status
- **WHEN** The `extract_items` function is invoked
- **THEN** An `ExtractedItem` is created with properties parsed from the text
- **AND** The item's type and status are successfully validated against the schema

#### Scenario: ExtractItemWithUnknownType
- **GIVEN** A source text describing a work item with a type not present in the ProjectSchema
- **WHEN** The `extract_items` function is invoked
- **THEN** The system attempts to resolve the type to a known canonical value via `resolve_type`
- **AND** If resolution fails, the item may be discarded or flagged as invalid

#### Scenario: ExtractHierarchicalItems
<!-- openlore-test: tags=smoke (auto) -->
- **GIVEN** A source text describing a parent task and a child sub-task
- **WHEN** The `extract_items` function is invoked
- **THEN** Two `ExtractedItem` instances are created
- **AND** The child item's `parent_id` property is set to the identifier of the parent item
